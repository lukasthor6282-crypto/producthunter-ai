# Roadmap do Produto

## Principio

O ProductHunter AI deve evoluir em etapas pequenas, mantendo o produto funcional a cada fase.

Prioridades:

- recomendacao util;
- explicacao clara;
- dados confiaveis;
- experiencia premium;
- arquitetura preparada para APIs oficiais;
- aprendizado de Machine Learning durante o desenvolvimento.

## Fase 1: MVP Funcional

Status: em andamento.

Entregas:

- backend FastAPI;
- frontend React;
- dados simulados realistas;
- ranking de recomendacoes;
- score de oportunidade;
- score de risco;
- simulador de lucro;
- dashboard;
- Laboratorio de IA;
- documentacao inicial.

Fora do escopo:

- login;
- pagamento;
- scraping;
- dados reais;
- multiusuario.

## Fase 2: Melhorias de Produto

Objetivo: deixar o MVP mais convincente para validacao com usuarios.

Entregas:

- historico local de recomendacoes;
- comparacao entre produtos;
- filtros avancados;
- exportacao CSV;
- favoritos;
- ajustes de perfil salvos no navegador;
- mensagens de explicacao mais personalizadas;
- melhoria de responsividade.

## Fase 3: Dados Reais Controlados

Objetivo: iniciar integracao com fontes permitidas.

Entregas:

- camada de providers por marketplace;
- cache de respostas;
- logs de origem dos dados;
- normalizacao de dados reais para o schema interno;
- atualizacao periodica;
- painel de qualidade dos dados.

Cuidados:

- usar APIs oficiais;
- respeitar rate limits;
- revisar termos de uso;
- evitar scraping proibido;
- nao armazenar dados pessoais sem necessidade.

## Fase 4: Conta, Assinatura e Produto Comercial

Objetivo: transformar o MVP em SaaS.

Entregas:

- autenticacao;
- workspace;
- planos;
- limite de uso;
- pagamentos;
- onboarding;
- billing;
- tela de conta;
- suporte basico.

## Fase 5: Inteligencia Avancada

Objetivo: melhorar previsoes com dados historicos reais.

Entregas:

- treino com vendas reais;
- validacao temporal;
- modelos por marketplace;
- recomendacao por objetivo;
- alertas de tendencia;
- monitoramento de drift;
- explicabilidade por recomendacao;
- testes A/B.

## Fase 6: Plataforma

Objetivo: criar ecossistema para times e integracoes.

Entregas:

- API publica;
- webhooks;
- times e permissoes;
- integracoes com lojas;
- integracoes com ads;
- relatorios automaticos;
- carteira de produtos monitorada;
- benchmarks de nicho.

## Ideias Futuras

- Radar de produtos emergentes.
- Score de viralizacao para TikTok Shop.
- Simulador de estoque.
- Calculadora de comissao para afiliados.
- Analise de criativos.
- Sugestao de publico-alvo.
- Sugestao de copy.
- Alertas de queda de margem.
- Analise de sazonalidade.
